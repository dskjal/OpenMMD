# アニメーションループ境界における剛体の貫通を修正

## 概要

- ループラップ、シーク、巻き戻しを物理演算の不連続として扱います。
- FK/IKが再計算された後、古い再計算前の行列ではなく、ループ開始時の最終ポーズから剛体を再同期します。
- 不連続に対してハードリセットポリシーを適用します。速度をゼロにし、蓄積されたモーション状態をクリアしてから、シミュレーションを再開します。


## 主な変更点

- source/model-manager.js において、更新順序を animationController.update() -> recomputeBoneMatrices() -> solveIk() -> 不連続同期 -> physicsEngine.update() -> 最終行列書き込み/デバッグ更新に変更します。不連続同期は、animationController.jumped が true の場合にのみ実行され、IK後の `worldMatrix` を使用する必要があります。
- source/physics.js において、現在の「ポーズリセット」動作を、不連続性に対する明示的なポーズ同期ルーチンに置き換えます。resetModel() は、パブリック名を変更しない場合はそのまま残しますが、そのコントラクトは以下のようになります。すべての剛体を現在のシーンポーズにテレポートし、重心とモーションステートの変換を更新します。線形/角速度をゼロにし、力をクリアし、Ammo がこれらのメソッドを公開している場合は補間変換/速度をリセットします。ボディをアクティブ化し、これらのフックが存在する場合は AABB/broadphase の状態を更新します。
- source/physics.js において、現在のフレームごとに justSynchronized フラグを追加します。_preSimulation() は、同じフレーム上の古いポーズから追いつき速度を生成してはなりません。不連続性同期後、運動学的ターゲットのデルタはゼロとして扱われます。
- AnimationController.jumped で既に検出されているすべてのケース（ループラップ、後方シーク、巻き戻し、大きなフレームジャンプ）に対して、ハードリセットポリシーを維持します。

これらのイベント間では、運動量を保持しません。

## テスト

- 大きなルートトランスレーションループに対する回帰テストを追加します。このテストでは、ループ後の最初の物理ステップが大きな運動学的速度を生成せず、リジッドボディをループ前のポーズに残さないことを確認します。
- ModelManager.update() のシーケンステストを追加します。このテストでは、不連続同期が FK/IK 再計算後に実行され、更新されたワールドマトリックスを読み取り、前のフレームマトリックスを読み取りません。
- 物理リセットテストを追加します。このテストでは、同期ルーチンが線形/角速度と力をクリアし、モーション/補間変換を更新し、利用可能な場合は AABB/ブロードフェーズ更新フックを呼び出すことを確認します。


## 前提条件

- 選択された動作はハードリセットです。不連続性は、運動量の維持よりも安定性と非貫通を優先します。
- boneIndex === -1 のボディは、別途バグが確認されない限り、作成者によるリセット動作を維持します。
- jumped が false の場合の通常の連続再生は変更されません。
