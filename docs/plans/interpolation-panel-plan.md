# 補間曲線操作パネル追加プラン

## 目的

`index.html` のモーションセクションに、VMD の補間曲線を編集できる「補間曲線操作パネル」を追加する。

レイアウトは[添付画像](./interpolation-panel.jpg)の構成を参考にし、以下を満たす。

- 左側に補間曲線の表示エリアを置く
- 右側に回転対象の選択、`自動設定`、`コピー`、`ペースト`、`線形補間` をまとめる
- カンバス上での有効操作はハンドルのドラッグのみ
- 曲線はアンチエイリアスを効かせて滑らかに表示する
- ベジェ曲線のアルゴリズムは `source/animation.js` の既存実装を利用する

## 前提

- MMD の補間は 2 次元のベジェ制御点として扱う
- デフォルトのハンドル値は `(0.2, 0.2)` と `(0.8, 0.8)` とし、これは線形補間を表す
- 補間曲線の編集対象はモーション編集 UI の一部として扱う
- 既存のデータ構造や保存形式を壊さない

## 画面構成

### レイアウト方針

画像に合わせて、横長の小さなパネルとして実装する。

- 左: 補間曲線のプレビューカンバス
- 右: 操作群
  - 回転種別の切り替え
  - `自動設定`
  - `コピー`
  - `ペースト`
  - `線形補間`

### 見た目の要件

- カンバス内に 0..1 の正規化座標系を表示する
- 枠線、グリッド、ハンドル、制御線、補間曲線を明確に区別する
- 現在のハンドル位置が視覚的に分かるようにする
- 線形補間は対角線として即座に識別できるようにする

## 実装方針

### 1. UI 追加

`index.html` のモーションセクションへパネルを追加する。

- 既存のモーション関連 UI と同じ階層に配置する
- 画面幅が狭い場合も崩れにくい flex または grid ベースのレイアウトにする
- 画像のように「左カンバス + 右操作」の 2 カラムを基本形にする

### 2. 状態管理

補間曲線編集用の状態を明確に持つ。

- 対象ボーンまたは対象キー種別ごとの補間値
- 4 点の制御点
  - 開始点は固定 `(0, 0)`
  - 終了点は固定 `(1, 1)`
  - 中間ハンドル 2 点を編集対象にする
- 編集中のドラッグ状態
- コピー・ペースト用の一時バッファ

### 3. ベジェ曲線の再利用

`source/animation.js` にあるベジェ処理を、描画プレビューと値計算の両方で再利用する。

- 補間の評価ロジックを重複実装しない
- UI から変更された制御点を既存のベジェ関数に渡せる形に整える
- 必要であれば、描画用のサンプリング関数を薄く追加する

### 4. カンバス描画

曲線プレビューは 2D カンバスで描画する。

- グリッド背景を薄く描く
- 基準対角線を描く
- 制御点と制御線を描く
- 補間曲線を太めの線で描く
- ハンドルはドラッグしやすいサイズの円または四角で描く

### 5. アンチエイリアス

曲線を美しく表示するため、カンバス描画にアンチエイリアスを導入する。

実装方針は以下を優先する。

- `devicePixelRatio` を使って高解像度描画する
- CSS サイズと内部解像度を分離する
- 線幅や座標を整数ピクセルに固定しすぎず、滑らかなサブピクセル描画を許可する
- 必要なら補間曲線を十分な分割数でサンプリングして描く

補足:

- 1 つの線分で無理に近似するより、複数サンプルで描いた方が見た目が安定する
- UI の小さなパネルでは、描画解像度の確保が最優先になる

### 6. 操作制約

カンバスで許可する操作はハンドルのドラッグのみとする。

- 背景のパンやズームは実装しない
- クリックによる即時移動は不要
- キーボード操作は不要
- ハンドルをカンバス外へ出さないようにクランプする
- 左右のハンドルが逆転しないように必要最低限の制約を設ける

### 7. 補助機能

右側のボタン群は次の役割にする。

- `自動設定`
  - 選択中データや既定ルールに基づき補間値を自動生成する
- `コピー`
  - 現在の補間値をクリップボード用の内部バッファへ退避する
- `ペースト`
  - 退避済み補間値を現在の対象へ適用する
- `線形補間`
  - `(0.2, 0.2)` と `(0.8, 0.8)` に戻す

## 実装手順

1. `index.html` にパネルの HTML を追加する
2. パネル用の CSS を追加し、画像に近い横並びレイアウトを作る
3. 補間曲線用の状態管理を UI 側に追加する
4. `source/animation.js` のベジェ実装を呼び出せるよう整理する
5. 2D カンバス描画を実装する
6. `devicePixelRatio` 対応とサンプリング描画でアンチエイリアスを有効にする
7. ハンドルドラッグだけを許可する入力処理を追加する
8. コピー・ペースト・線形補間・自動設定を接続する
9. 既存のモーション編集フローと統合する
10. テストと目視確認を行う

## テスト観点

### 描画

- パネル表示時に曲線が崩れず、ジャギーが目立たない
- 線形補間のときは対角線が正しく表示される
- ハンドル位置の変更が即時に反映される

### 操作

- ハンドル以外の領域を操作しても値は変化しない
- ドラッグ中にカンバス外へ出ても値が破綻しない
- コピー後のペーストで同じ補間値が復元される
- 線形補間ボタンでデフォルト値へ戻る

### 互換性

- 既存の VMD 読み書きに影響しない
- `source/animation.js` の既存の補間計算結果と一致する
- モーション編集セクションの他 UI と干渉しない

## リスクと対策

- リスク: 小さなカンバスで曲線が読みにくい
  - 対策: 高 DPI 描画と十分な線幅を使う
- リスク: ベジェ評価ロジックの重複で不整合が出る
  - 対策: `source/animation.js` の実装を単一の参照元にする
- リスク: ドラッグ操作が誤検知される
  - 対策: ハンドルのヒット領域を大きめにする

## 完了条件

- モーションセクションに補間曲線操作パネルが追加されている
- カンバスで補間曲線を確認できる
- ハンドルをドラッグして補間値を変更できる
- 線形補間、コピー、ペースト、自動設定が機能する
- 曲線表示がアンチエイリアスされて見やすい
