 # Custom Bone Circle Picking

  ## Summary

  source/model-debug-draw.js で表示している 全ての親 / センター / 下半身 の円形カスタム形状を、左クリックで直接選
  択できるようにする。実装は source/gizmo.js の円ヒット判定ロジックを再利用し、通常のボーン近傍クリック判定より先
  に評価する。

  ## Key Changes

  - source/gizmo.js
      - 既存のリング判定ロジックを、通常ギズモ以外からも使える形で切り出して export する。
      - 新しい公開関数は「中心座標・法線・半径・厚み・レイ」を受けてヒット有無と距離を返す形にする。
      - 既存の pickGizmo はその公開関数を内部利用する形に揃え、判定式を二重管理しない。
      - 追加コードは JSDoc を付ける。
  - source/model-debug-draw.js
      - 円表示に使っている仕様を定数化する。
      - 対象は 3 つに固定する。
          - 全ての親: オフセット [0, 0, 0], 半径 2.5
          - センター: オフセット [0, 4, 0], 半径 2.5
          - 下半身: オフセット [0, -2, 0], 半径 2.5
      - 描画時に使う値とピック時に使う値がずれないよう、円定義を共有できる小さな helper を追加する。
      - ピックに必要な幾何は Y 軸法線の水平リングとして扱う。
  - source/renderer-interaction.js
      - pickSceneElement のアクティブインスタンス判定内で、通常ボーン選択より前にカスタム円のヒットテストを追加す
        る。
      - フローは以下で固定する。
          1. 物理剛体ピック
          2. アクティブインスタンスの通常ギズモピック
          3. アクティブインスタンスのカスタム円ピック
          4. IK ターゲット / 通常ボーンの 2D 距離ピック
          5. 非アクティブインスタンスの既存ピック
          6. AABB ピック
      - カスタム円がヒットしたら、その対応ボーン index を selectedBoneIndex に設定し、selectedTargetIndex と
        selectedRigidbodyIndex は -1 にする。
      - 複数リングがヒットした場合は、レイ上の距離が最小のものを優先する。
      - 非表示扱いのボーンは既存ルールに合わせて選択対象から除外する。

  ## Test Plan

  - アクティブモデルで 全ての親 の円をクリックすると、そのボーンが選択される。
  - アクティブモデルで センター の円をクリックすると、そのボーンが選択される。
  - アクティブモデルで 下半身 の円をクリックすると、そのボーンが選択される。
  - 円の近くに通常ボーンがあっても、円線上をクリックした場合は対応カスタムボーンが優先される。
  - 円に当たらない場合は既存の IK ターゲット / 通常ボーン選択が従来通り動作する。
  - 通常の回転/移動ギズモが表示されている状態でも、既存の pickGizmo 動作は壊れない。
  - ボーンが存在しないモデルでは何も追加選択されない。

  ## Assumptions

  - カスタム円は現在の描画実装通り、水平面上のリングとしてピックする。
  - 対象ボーンは model.customRigBones と実際の円描画対象の 3 ボーンに限定する。
  - カスタム円ピックは表示上もアクティブインスタンスにしか存在しないため、非アクティブインスタンスには追加しない。
  - 厚みしきい値は gizmo.js 側のリング判定と同等スケール感で扱い、必要なら定数 1 箇所で調整できるようにする。
