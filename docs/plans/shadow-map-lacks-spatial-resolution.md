# シャドウマップの空間分解能不足の解決方法

影の輪郭がギザギザになる原因の大半は、**シャドウマップの絶対解像度不足ではなく、光源視点の1 texel がシーン上でどれだけ広い面積を覆っているか**です。2K でも、ライト用の投影範囲が広すぎると普通に粗くなります。

効きやすい順に挙げると、次の対策が本命です。

**1. ライトの投影範囲を絞る**
Directional light なら、影を落とす範囲に対して light frustum を最小化します。
遠くまで広く取ると 2K でも texel density が落ち、輪郭が段差状になります。
特に「カメラに見えている範囲だけ」を tight fit するのが有効です。

**2. Cascaded Shadow Maps にする**
Directional light ではこれが最も定番です。
近距離は高解像度、遠距離は低解像度に分けるので、輪郭のジャギーがかなり減ります。
1枚2Kより、たとえば 4 枚のカスケードで分配した方が実効品質は上がります。

**3. シャドウマップの texel snapping を入れる**
light frustum をカメラ移動に合わせて連続的に動かすと、ギザギザというより「ちらつき」や「段差の変化」が目立ちます。
ライトの投影行列を texel 単位でスナップして、シャドウの安定性を上げます。
Moment Shadow Mapping は平滑化されやすいですが、投影の不安定さは別問題です。

**4. Moment Shadow Mapping の前段でフィルタリングを適切に使う**
MSM は「滑らかに見せる」ための手法ですが、**元の深度分布が粗いと輪郭の形自体は改善されません**。
結果として「少しぼけたギザギザ」になります。
必要なら、保存している moment に対して適切な mip / box filter / separable filter を使います。

**5. 受け側・ライト側の bias を調整する**
bias が弱いとエッジ付近で不安定になり、輪郭が汚く見えます。
ただし bias を強くしすぎると、今度は影が浮くので注意です。
使うなら

* depth bias
* slope-scale bias
* normal offset bias
  を分けて調整します。

**6. 近距離だけ別手法を足す**
カメラ近くの接地影は、シャドウマップだけだと厳しいことがあります。
その場合は

* contact shadows
* screen-space shadow
* ray query 系の補助
  を併用すると、見た目がかなり安定します。

実務的には、次の順で試すのが効率的です。

1. **ライト投影範囲を狭める**
2. **Cascaded Shadow Maps 化する**
3. **texel snapping を入れる**
4. **bias を再調整する**
5. **必要なら近距離補助を追加する**

結論として、**2K だから十分とは限らず、まず light frustum の設計を見直すのが最優先**です。
Moment Shadow Mapping 自体は「影を滑らかにする」には効きますが、**輪郭の空間分解能不足**は解決できません。
