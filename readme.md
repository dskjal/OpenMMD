# ローカルでの実行方法
### 必要なアプリ
- Git
- Node.js

### コマンド

1. ソースのダウンロード
> git clone https://github.com/dskjal/OpenMMD  
> cd openmmd

2. 依存関係のインストール
> npm install

3. サーバー起動

> npm start

4. `http://localhost:3000/` にアクセス。

### API

API のリストは `docs/specification/api-specification-ja.md` を参照。

サンプルは `./codex/skills/posing/down-center.ps1` を参照。

### 動画書き出し

動画書き出しの背景透過は `WebM` と `MKV` のみ対応です。

動画設定の `背景を透過` を有効にすると、対応形式では背景の空き部分を alpha 付きで書き出します。`MP4` と `MOV` ではこの設定は無効になります。

### GitHub Pages

`index.html` は GitHub Pages 上では静的モードで動作し、API 機能は無効になります。

公開用の静的成果物は `npm run build:pages` で `dist/` に生成します。GitHub Actions の `Deploy GitHub Pages` ワークフローが `dist/` を Pages に配信します。

## OpenMMD の使い方
### ファイルのロード
右上をクリックするか、zip ファイルをウインドウにドラッグ＆ドロップ
zip/フォルダ内に json や vpd があると自動的に読み込まれます。

*pmd, pmx のドラッグ＆ドロップは非対応です*。ウェブブラウザのセキュリティ機能でファイルの単体ドロップにはパス情報がつかないため、テクスチャが読み込まれません。pmd, pmx が入ったフォルダ/zip をドラッグ＆ドロップしてください。

### 保存

#### vpd
vpd ファイルには **`選択されているボーンのみ保存`** されます。

vpd ファイルに保存されるポーズは `vmd のポーズ + ユーザーが入力したポーズ` が保存されます。

### 編集するキャラの切り替え
ビューポート上でキャラをクリック。

### チャイルド
アクセサリをつけたり、武器を持たせたりするのに使います。

1. チャイルドを設定するボーン（武器やアクセサリ）を選択する
1. モデルと、親にしたいボーンを選択
1. `逆補正を解除`でボーン位置に移動します。

### キーの打ち方
赤丸をクリックするとキーを打ちます。

### VRM `下半身`ボーンの注意点

VRM で vmd を再生できるようにするために、VRM の仕様にはない`下半身`ボーンが挿入されています。**`下半身`ボーンにキーを打つと、作成した vrma に互換性がなくなります**。

### ボーン
#### IK の追加
IK の末端ボーンに IK を追加します。

腕に IK を設定する場合、VRM の場合は `ForeArm` に IK を追加、pmx の場合は`ひじ`に IK を追加します。

pmx の場合、IK 操作するボーンにチャイルドを設定し、`センター`を親ボーンにすると便利です。

##### 腕が２分割されている場合

タイムラインから`捩`ボーンを選択して、IK を設定します。ターゲットボーンを`手首`に設定すると大抵うまくいきます。

チェーン数を４にして、`捩`ボーンに IK lock を設定します。

#### IK/FK 切り替え

IK/FK 切り替えスイッチはありません。

IK 時にボーンにローカル回転が直に入力されます。なので、IK 操作ボーンではなく、腕や脚にボーンにキーを打つとそのままローカル回転にキーが打てます。アニメーション時には IK を無効にします。

### パフォーマンス制限

- アンチエイリアシングを`なし`や`FXAA`にする
- レンダリング FPS を下げる
- ポストエフェクトを切る

### アンチエイリアシング
FXAA で画像が若干ぼやけるのは正常です。FXAA は DPI の高いモニターでパフォーマンスを優先するときに使います。

FXAA と色収差は同時に使えません。色収差が無効になります。

### 影
|問題|対処法|
|:---|:---|
|影にギザギザのモアレが出る|バイアスを上げるか、シャドウマップ解像度を上げる。グリッドを非表示にする|
|影の輪郭にジャギーがでる|シャドウマップ解像度を上げる。グリッドを非表示にする|
|カメラに映っていないオブジェクトの影が出ない|Far 自動のチェックを外し、手動調整する|
|細かい部分に影が出ない|バイアスを上げると細かい部分の影が消える。バイアスを下げて、シャドウマップ解像度を上げる。アンビエントオクルージョンを使う|

### 被写界深度

赤いスポイトマークをクリックした後ビューポートをクリックすることで、焦点位置を設定できます。キャンセルはもう一度赤いスポイトをクリック。

|アルゴリズム|説明|
|:---|:---|
|Fast|速いが低品質。プレビュー用だが、ボケの量が少ないなら最終出力にも使える|
|Depth-Aware Gather|深度差とサンプル側の錯乱円とを見ながら周辺画素をサンプルする。品質と計算負荷とのバランスが取れている|
|Thin Lens Multisample|他の二つより処理が重いが、高品質。モデルに接近して大きなボケが発生する場合におすすめ。ミドルレンジ以上の GPU なら常用可能|

### 色温度

スポイトは白色点を設定します。ライトの色を設定した後、スポイトで本来白であるべき部分をクリックすると、色温度を自動設定します。

## HDR

### 設定
表示プリセットで HDR を選択するか、以下の設定にする。

1. `描画設定` > `カラーマネジメント` > `ビュー変換` > を `ACES 2.0` にする
1. マテリアルタブでマテリアルに HDR 対応シェーダを設定する
    - glTF Shader
    - MToon Shader

### tips

- ライトの強さを１以上にする。これでブルームを効かせることができる
    - 影にブルームを効かせたくない場合は、ブルーム影乗算を上げる
- コントラストを下げたいとき
    - hdr 環境テクスチャの明るさを上げる
    - 描画設定タブの影のパワーを下げる
- ポストエフェクトのガンマの数値を下げてルックを調整する。目安は 0.3 程度
- 影のパワーを上げて影の色を濃くする
- 肌の`拡散色`に赤を混ぜる
- 金属の反射を強調したいときは、`粗さ`を下げて、環境テクスチャの明るさを上げる

## シェーダの書き方

`docs/custom-shader.md` を参照。

model json でカスタムシェーダを指定できる。zip/フォルダ内に json と wgsl とを入れておくと、zip/フォルダをドラッグ＆ドロップしたときに自動的に読み込まれる。

``` json の例
{
    "type": "model",
    "model-name": "初音ミク",
    "material": {
        "cloth": {
            "material-shader-select": "my-shader.wgsl",
            "material-visible": 1,
            ...
        }
    }
}
```

model JSON の保存はマテリアルタブの `Model JSON を保存` からできる。

### シェーダのリロード

ローカルで動作させている場合は `source/shaders/custom-shaders/` にシェーダを配置し、`manifest.json` にシェーダのパスを記載すると、シェーダ選択セレクトボックス右のリロードアイコンで、アプリをリロードせずにシェーダをリロードできる。

### HDR プリセット

`display-preset.js` で設定のプリセットが変更できる。
